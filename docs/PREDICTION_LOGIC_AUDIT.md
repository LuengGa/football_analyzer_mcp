### 2.2 泊松模型深度分析 ✅

**核心公式链**:
```
home_attack × away_defense × league_avg × home_advantage → lambda_home
away_attack × home_defense × league_avg → lambda_away
lambda_home, lambda_away → Poisson比分矩阵 → SPF/BF/ZJQ/BQC概率
```

**与官方规则的精确对应**:

| 官方规则要素 | 泊松模型实现 | 契合度 |
|-------------|-------------|--------|
| SPF(胜/平/负) | 矩阵聚合 h>a / h==a / h<a | ✅ 精确 |
| BF(31个比分) | 矩阵直接提取 P(h,a) | ✅ 精确 |
| BF(胜其他) | 尾部概率 + 未列出的主胜比分 | ✅ 精确 |
| BF(平其他) | 尾部概率 + 未列出的平局比分 | ✅ 精确 |
| BF(负其他) | 尾部概率 + 未列出的客胜比分 | ✅ 精确 |
| ZJQ(0-7+) | 矩阵聚合 total=h+a | ✅ 精确 |
| BQC(半全场) | 半场矩阵×全场矩阵×相关性修正 | ✅ 精确 |
| RQSPF(让球) | lambda_home-handicap 重建矩阵 | ✅ 精确 |

**Dixon-Coles修正** (关键亮点):
```python
# 修正低比分相关性偏差
if h==0 and a==0: tau = 1 - home_exp * away_exp * rho
if h==0 and a==1: tau = 1 + home_exp * rho
if h==1 and a==0: tau = 1 + away_exp * rho
```
这解决了0:0、0:1、1:0这三种最常见比分在独立泊松假设下的概率偏差，**直接提升了BF和SPF的预测精度**。

### 2.3 Elo评级深度分析 ✅

**与官方规则的对应**:

| 官方规则要素 | Elo实现 | 契合度 |
|-------------|---------|--------|
| 主场优势 | +35分Elo加成 | ✅ 合理 |
| 联赛差异 | 27个联赛基准Elo(英超1650→中甲1200) | ✅ 精确 |
| 比赛重要性 | K因子:友谊赛20~世界杯48 | ✅ 精确 |
| 新球队 | 联赛基准+确定性扰动 | ✅ 合理 |
| 大胜加权 | 胜+min(0.3, diff*0.1) | ✅ 精确 |
| 均值回归 | 偏离>100分且场次<5时回归2%/场 | ✅ 精确 |

**Elo→胜平负概率转换**:
```python
draw_factor = 0.26  # 足球典型平局率
win = E * (1 - draw_factor)
draw = draw_factor * (1 - |E-0.5| * 2 * dampening)
lose = (1-E) * (1 - draw_factor)
```
这个转换公式考虑了足球平局率高的特点，**与竞彩SPF玩法的三选项结构完美对应**。

## 四、发现的问题与改进建议

### ✅ 返还率实现已正确（联网核对确认）

**官方规则** ([来源](https://www.sporttery.cn/help/273604.html)):
> 竞彩使用**固定赔率**，奖金计算公式为：`单注奖金 = 2元 × 所选比赛场次的固定奖金额连乘`

**关键结论**: 
- 竞彩的固定赔率**已经内含返还率**
- **不需要再乘70%或65%**
- 代码中 `return_rate=0.70` 的使用场景是**价值投注判断**（计算市场隐含概率时需要调整），而非奖金计算
- `constants.py` 中的 `PLAY_RETURN_RATE` 用于估算理论赔率，非错误

**之前审计报告的判断错误，已纠正。**

### ✅ 比分选项已补全（联网核对确认）

**官方规则** ([来源](https://www.sporttery.cn/help/2968.html?gid=9)):
> 比分玩法共设置**31种**比分结果选项：
> - 主队获胜：1:0, 2:0, 2:1, 3:0, 3:1, 3:2, 4:0, 4:1, 4:2, **5:0, 5:1, 5:2**, 胜其它（共13个）
> - 平局：0:0, 1:1, 2:2, 3:3, 平其它（共5个）
> - 主队落败：0:1, 0:2, 1:2, 0:3, 1:3, 2:3, 0:4, 1:4, 2:4, **0:5, 1:5, 2:5**, 负其它（共13个）

**已修复**:
- `constants.py` 中 `PLAY_OPTIONS["BF"]` 已补全 5:0, 5:1, 5:2, 0:5, 1:5, 2:5
- `OFFICIAL_BF_SCORES`, `BF_HOME_WINS`, `BF_AWAY_WINS` 已同步更新

### 🟡 问题3: BQC半场比率缺乏联赛差异化 [中等]

**位置**: `bqc_plugin.py`

当前所有联赛使用统一的 `ht_ratio=0.45`（半场进球占全场45%）。

**实际情况**: 不同联赛的半场进球比率有差异：
- 德甲: ~48% (上半场进球较多)
- 意甲: ~42% (上半场进球较少)
- 英超: ~45% (中等)

**修复建议**: 在联赛基准数据中添加 `ht_ratio` 字段，BQC插件按联赛动态获取。

### 🟢 问题4: ZJQ独立泊松回退过于粗糙 [低]

**位置**: `zjq_plugin.py`

当无比分矩阵时，ZJQ使用四档预设概率分布（<1.5球/1.5~2.5球/2.5~3.5球/>3.5球），粒度较粗。

**修复建议**: 使用泊松分布直接计算 `P(total=k) = PoissonPMF(k, lambda_home+lambda_away)` 替代预设分布。

### 🟢 问题5: 矛盾检测缺少 BF↔BQC 直接检测 [低]

**位置**: `jingcai_rule_engine.py`

当前有 SPF↔BF、BF↔ZJQ、SPF↔RQSPF、BQC↔SPF 四组矛盾检测，但缺少 BF↔BQC 的直接检测。

**修复建议**: 添加 BF比分方向与BQC全场结果的一致性检测。

### 🟢 问题6: engine.py 与 models.py 存在重复 [低]

**位置**: `analysis/engine.py` vs `analysis/models.py`

两个文件都定义了 `StatisticalEngine` 类，但：
- `engine.py` 版本：仅泊松+Elo，功能简化
- `models.py` 版本：泊松+Elo+xG，功能完整

**修复建议**: 统一使用 `models.py` 版本，将 `engine.py` 中的独特功能（风险信号检测、报告生成）迁移到 `models.py`。
