**版本**: v5.0 深度评审

**基于**: 前4轮修复(60个问题) + 第5轮深度评审(玩法专属逻辑/数据集成/规则消化)

## 一、评审总论：当前架构的核心问题

经过对6种玩法官方规则、分析逻辑架构、数据获取工具的深度交叉审查，发现当前MCP的核心问题**不是代码bug**（前4轮已修复），而是**分析深度的结构性缺陷**：

> **当前状态**：6种玩法共享同一个泊松模型 → 从同一组λ值出发 → 各自做简单变换 → 输出概率  
> **应有状态**：每种玩法有独特的概率结构、独立的数据需求、专属的分析维度

### 1.1 六大玩法的本质差异（规则层）

| 维度 | SPF | RQSPF | BF | ZJQ | BQC | 混合过关 |
|------|-----|-------|----|----|-----|---------|
| **竞猜对象** | 胜负方向 | 让球后胜负方向 | 具体比分 | 总进球数 | 半场+全场组合 | 多玩法组合 |
| **选项数** | 3 | 3 | 31 | 8 | 9 | N/A |
| **最大串关** | 8 | 8 | **4** | **6** | **4** | 木桶原则 |
| **概率空间** | 3元简单 | 3元(条件) | 31元高维 | 8元有序 | 9元条件联合 | 跨玩法约束 |
| **核心难度** | 低 | 中(需盘口判断) | **极高** | 中 | **高** | 高(约束满足) |
| **独特分析维度** | 主客场优势 | 盘口深度/让球合理性 | 比分分布形态 | 进球期望区间 | 半场节奏/逆转概率 | 玩法互补/矛盾 |

### 1.2 当前分析逻辑的耦合/解耦诊断

```
                    PoissonModel.predict()
                           │
              ┌────────────┼────────────┐
              │            │            │
         win/draw/   home_exp      score_matrix
         lose_prob   away_exp      (full_matrix)
              │            │            │
         ┌────┘       ┌────┘       ┌────┘
         │            │            │
        SPF      RQSPF/BF/ZJQ     BQC
        (直接用)   (从λ重建)    (从矩阵聚合)
```

**问题诊断**：

| 问题编号 | 严重度 | 问题描述 |
|---------|--------|---------|
| D-01 | 🔴 关键 | **SPF只用聚合概率，完全忽略了比分层信息**。SPF的"胜"可以由1:0、2:1、3:0等多种比分构成，不同比分构成反映不同比赛特征，当前SPF分析完全丢失了这个维度 |
| D-02 | 🔴 关键 | **RQSPF重复计算且与PoissonModel不一致**。PoissonModel已经计算了`home_win_prob_adjusted`，但RQSPF完全忽略，自己从λ重建矩阵。两套计算可能产生矛盾结果 |
| D-03 | 🔴 关键 | **BF的比分概率仅靠泊松模型，缺少历史校准**。86,040场历史数据中有完整的比分频率分布，但BF分析从未使用这些真实数据进行校准 |
| D-04 | 🟠 严重 | **ZJQ缺少大小球市场数据交叉验证**。`get_market_odds`返回的`over_under`数据（多机构大小球盘口）是ZJQ的天然对照，但ZJQ分析从未消费这些数据 |
| D-05 | 🟠 严重 | **BQC的半场模型过于简化**。HT_RATIO=0.45是固定常数，但不同联赛/球队的半场进球比例差异很大（如意甲半场进球比例偏低，德甲偏高） |
| D-06 | 🟠 严重 | **历史数据(86,040场)与分析流水线完全断裂**。`lottery_historical_analysis`使用hardcoded模拟数据，而非真实历史数据 |
| D-07 | 🟡 中等 | **混合过关矛盾检测不完整**。缺少BQC与其他玩法的矛盾检测（如BQC"负-胜"与SPF"主胜"的矛盾） |
| D-08 | 🟡 中等 | **木桶原则实现存在逻辑重复**。`_apply_bucket_principle()`和`_validate_plan_rules()`中有两份独立的木桶检查，行为不一致 |
| D-09 | 🟡 中等 | **赔率字典键名多态兼容逻辑分散**。每个玩法方法内部都有多套赔率键名兼容（如`home_win`/`win`/`spf_home`），未统一抽取 |
| D-10 | 🟢 低 | **PlayType枚举映射字典重复3次**。`name_to_type`和`str_to_play_type`在mixed_parlay.py中重复定义 |

## 二、六大玩法深化方向（规则×数据×模型 三维交叉）

### 2.1 SPF（胜平负）— 深化方向

**官方规则特征**：
- 竞猜全场90分钟胜负平，不让球
- 3个选项，最基础的玩法
- 可单关，最多8关

**当前分析深度**：⭐⭐☆☆☆（仅用聚合概率）

**深化方向**：

| 方向 | 具体内容 | 数据来源 |
|------|---------|---------|
| **比分构成分析** | 将SPF"胜"拆解为具体比分概率分布（1:0占多少、2:1占多少等），判断胜的"质量" | 泊松比分矩阵 |
| **主客场差异化** | 利用86,040场历史数据，按联赛统计主胜/平/客胜的真实频率，作为先验校准泊松概率 | historical_tools |
| **欧指对比分析** | 将泊松概率与国际市场欧指均值(consensus)对比，发现市场与模型的分歧 | get_market_odds |
| **凯利指数偏离** | 利用多机构凯利指数，判断市场对哪个方向的共识最强 | get_market_odds |

**优化项**：
```
SPF-01: 新增 _decompose_spf_by_scores() 方法
       — 将SPF概率拆解为比分构成，输出如 "胜(45%) = 1:0(15%) + 2:1(12%) + 2:0(10%) + ..."
       
SPF-02: 新增 _calibrate_with_historical_prior() 方法
       — 用历史数据的主胜/平/客胜频率作为贝叶斯先验，校准泊松概率
       — 公式: P_posterior = P_poisson × P_historical / P_evidence
       
SPF-03: 新增 _compare_with_consensus() 方法
       — 将模型概率与欧指均值对比，输出分歧度指标
       — 分歧度 > 15% 时标记为"市场异常信号"
```

### 2.2 RQSPF（让球胜平负）— 深化方向

**官方规则特征**：
- 必须让球（整数），让球范围 -3 到 +3
- 2013年6月4日后独立为单独玩法
- 让球计算：主队得分 - 让球数 vs 客队得分
- 可单关，最多8关

**当前分析深度**：⭐⭐⭐☆☆（独立重建矩阵，但缺少盘口合理性判断）

**深化方向**：

| 方向 | 具体内容 | 数据来源 |
|------|---------|---------|
| **盘口合理性评估** | 判断竞彩让球数是否合理：与亚盘(Asian Handicap)对比，发现盘口偏差 | get_market_odds (asian_handicap) |
| **让球深度分析** | 让1球/2球/3球时，所需净胜球的历史达成率 | historical_tools |
| **让球灵敏度** | 分析让球数变化1球对胜平负概率的影响幅度 | 泊松模型 |
| **亚盘欧指统一视角** | 将让球胜平负概率与亚盘赔率统一到同一框架下分析 | get_market_odds |

**优化项**：
```
RQSPF-01: 新增 _evaluate_handicap_reasonableness() 方法
          — 将竞彩让球数与国际亚盘对比
          — 偏差 > 0.5球时标记为"盘口异常"
          — 输出：竞彩让-1 vs 亚盘让-0.75 → 偏差0.25球(合理)
          
RQSPF-02: 新增 _calculate_handicap_achievement_rate() 方法
          — 从历史数据统计：让N球时，主队净胜≥N球的概率
          — 按联赛/主客场分别统计
          
RQSPF-03: 新增 _handicap_sensitivity_analysis() 方法
          — 输出让球数从-3到+3的完整概率变化曲线
          — 找出"概率拐点"：让几球时胜平负概率最均衡
          
RQSPF-04: 修复RQSPF与PoissonModel的重复计算问题
          — 删除RQSPF中的独立矩阵重建
          — 改为消费PoissonModel的adjusted概率
          — 或者在PoissonModel中提供官方的let_ball接口
```

### 2.3 BF（比分）— 深化方向

**官方规则特征**：
- 31个选项（主胜13 + 平局5 + 客胜13）
- "胜其他"= 主队净胜4球及以上
- "平其他"= 4:4及以上平局
- "负其他"= 主队净负4球及以上
- 最多4关（难度最高的玩法之一）
- 可单关

**当前分析深度**：⭐⭐⭐☆☆（有泊松矩阵+尾概率补偿，但缺少历史校准）

**深化方向**：

| 方向 | 具体内容 | 数据来源 |
|------|---------|---------|
| **历史比分频率校准** | 用86,040场真实比分分布校准泊松概率 | historical_tools |
| **比分聚类推荐** | 不推荐单个比分，而是推荐比分区间（如"低比分组:1:0/1:1/2:1"） | 泊松矩阵 |
| **联赛特征比分** | 不同联赛有特征比分（如英超2:1常见，意甲1:0常见） | historical_tools |
| **比分与SPF/BQC的交叉验证** | 比分预测应与SPF方向、BQC半全场结果一致 | 内部交叉检查 |

**优化项**：
```
BF-01: 新增 _calibrate_scores_with_history() 方法
       — 从历史数据提取联赛级别的比分频率分布
       — 用贝叶斯方法校准泊松比分概率
       — 公式: P(score|league) ∝ P_poisson(score) × P_history(score|league)
       
BF-02: 新增 _cluster_scores_by_probability() 方法
       — 将31个比分按概率聚类为3-5组
       — 输出如: "高概率组(65%): 1:0/1:1/2:1/0:0" "中概率组(25%): 2:0/0:1/3:1" "低概率组(10%): 其余"
       — 推荐组而非单个比分，更实用
       
BF-03: 新增 _get_league_characteristic_scores() 方法
       — 从历史数据提取联赛特征比分Top5
       — 如: 英超→2:1/1:1/2:0/1:0/3:1, 意甲→1:0/0:0/1:1/2:1/2:0
       
BF-04: 新增 _cross_validate_with_spf() 方法
       — 检查比分预测的胜负方向是否与SPF一致
       — 如: SPF推荐"胜"，但BF最高概率比分是0:1 → 矛盾，需要警告
```

### 2.4 ZJQ（总进球）— 深化方向

**官方规则特征**：
- 8个选项：0/1/2/3/4/5/6/7+
- 7+ = 总进球7球及以上
- 最多6关
- 可单关
- 历史分布：2-3球最常见（约50%）

**当前分析深度**：⭐⭐⭐☆☆（有独立泊松计算，但缺少大小球交叉验证）

**深化方向**：

| 方向 | 具体内容 | 数据来源 |
|------|---------|---------|
| **大小球市场交叉验证** | 将ZJQ概率与国际大小球(Over/Under)盘口对比 | get_market_odds (over_under) |
| **历史进球分布校准** | 用历史数据校准总进球区间概率 | historical_tools |
| **联赛进球特征** | 不同联赛场均进球差异显著（如德甲~3.2球，意甲~2.5球） | historical_tools + LEAGUE_AVG_GOALS |
| **进球期望区间分析** | 不仅预测最可能进球数，还要给出置信区间 | 泊松分布 |

**优化项**：
```
ZJQ-01: 新增 _cross_validate_with_over_under() 方法
        — 将ZJQ概率与国际大小球盘口对比
        — 如: ZJQ预测总进球2.8球，大小球盘口2.5球 → 大球价值
        — 输出: "模型预期2.8球 vs 市场线2.5球 → 偏差0.3球(倾向大球)"
        
ZJQ-02: 新增 _calibrate_with_league_goals_dist() 方法
        — 从历史数据提取联赛级别的总进球分布
        — 与泊松预测对比，输出校准后的概率
        
ZJQ-03: 新增 _analyze_goal_expectation_interval() 方法
        — 输出总进球的置信区间
        — 如: "预期总进球2.8球, 68%置信区间[1.5, 4.1], 95%置信区间[0.3, 5.3]"
```

### 2.5 BQC（半全场）— 深化方向

**官方规则特征**：
- 9个选项：胜胜/胜平/胜负/平胜/平平/平负/负胜/负平/负负
- 第一个字=半场结果，第二个字=全场结果
- 最多4关
- 可单关
- 逆转（胜负/负胜）赔率最高

**当前分析深度**：⭐⭐⭐☆☆（有半场泊松+相关性修正，但HT_RATIO过于简化）

**深化方向**：

| 方向 | 具体内容 | 数据来源 |
|------|---------|---------|
| **联赛级半场比例** | 不同联赛半场进球比例差异大，不应使用固定HT_RATIO=0.45 | historical_tools (half_time_goals) |
| **历史半全场频率** | 86,040场有半场+全场结果，可直接统计9种组合的真实频率 | historical_tools |
| **逆转概率专项分析** | 负胜/胜负的逆转概率与球队实力差、主场优势的关系 | 泊松模型 + 历史数据 |
| **半场节奏分析** | 上半场进球时间分布（前15分钟/中15分钟/后15分钟） | — (需外部数据) |

**优化项**：
```
BQC-01: 新增 _get_league_ht_ratio() 方法
        — 从历史数据按联赛计算真实半场进球比例
        — 替换固定HT_RATIO=0.45为动态值
        — 如: 英超HT_RATIO=0.43, 德甲HT_RATIO=0.48, 意甲HT_RATIO=0.41
        
BQC-02: 新增 _calibrate_bqc_with_history() 方法
        — 从历史数据提取半全场9种组合的真实频率
        — 作为贝叶斯先验校准泊松BQC概率
        
BQC-03: 新增 _analyze_reversal_probability() 方法
        — 专项分析逆转概率（胜负/负胜）
        — 输入: Elo差距、主场优势
        — 输出: 逆转概率 + 与历史逆转频率的对比
        
BQC-04: 修复BQC相关性修正因子为动态计算
        — 当前: 固定 corr [1.0, 1.3] / [0.4, 0.8] / 0.85
        — 改进: 基于半场预期进球数动态计算相关性
        — 半场进球越少 → 不确定性越大 → 逆转概率越高
```

### 2.6 混合过关 — 深化方向

**官方规则特征**：
- 5种玩法可混合串关
- 同一场比赛不可选多个玩法（核心限制）
- 最大场次受木桶原则约束
- 不可单关

**当前分析深度**：⭐⭐⭐☆☆（有木桶原则+矛盾检测+5种策略，但不完整）

**深化方向**：

| 方向 | 具体内容 | 数据来源 |
|------|---------|---------|
| **BQC矛盾检测补全** | 当前缺少BQC与SPF/BF/ZJQ的矛盾检测 | 内部逻辑 |
| **玩法互补性评分** | 不只检测矛盾，还要量化互补性（如SPF主胜+ZJQ2-3球=高互补） | 内部逻辑 |
| **木桶原则逻辑统一** | 消除_validate_plan_rules与_apply_bucket_principle的重复 | 内部重构 |
| **玩法组合历史胜率** | 从历史数据统计不同玩法组合的实际命中率 | historical_tools |

**优化项**：
```
MIX-01: 补全BQC矛盾检测
        — BQC"负-胜"(半场负全场胜) vs SPF"主胜" → 矛盾(半场主负但全场主胜概率低)
        — BQC"胜-负"(半场胜全场负) vs SPF"主胜" → 矛盾
        — BQC"平平" vs ZJQ"0" → 一致(互补)
        
MIX-02: 新增 _calculate_play_complementarity_score() 方法
        — 量化两个玩法的互补性(0-1)
        — 互补性高 → 适合混合过关
        — 互补性低/矛盾 → 不建议组合
        
MIX-03: 统一木桶原则实现
        — 删除_validate_plan_rules中的重复检查
        — 仅保留_apply_bucket_principle作为唯一实现
        — _validate_plan_rules调用_apply_bucket_principle
        
MIX-04: 新增 _analyze_play_combination_history() 方法
        — 从历史数据统计不同玩法组合的命中率
        — 如: "SPF+ZJQ组合历史命中率45%, SPF+BF组合历史命中率12%"
```

## 三、数据集成优化（历史数据打通）

### 3.1 核心问题

当前86,040场历史数据与分析流水线**完全断裂**：

```
historical_tools (独立查询接口)
    ↕ 未连接
analysis_pipeline (实时分析流水线)
    ↕ 未连接
play_analysis (概率计算)
```

### 3.2 优化方案：历史数据校准层

```
                    ┌─────────────────────┐
                    │  HistoricalCalibrator │  ← 新增模块
                    │  (历史数据校准器)      │
                    └──────────┬──────────┘
                               │
              ┌────────────────┼────────────────┐
              │                │                │
        league_prior     score_prior      bqc_prior
        (联赛胜平负频率)  (联赛比分频率)    (半全场频率)
              │                │                │
              └────────────────┼────────────────┘
                               │
                    BayesianCalibration
                               │
                    ┌──────────▼──────────┐
                    │   PlayAnalyzer      │
                    │ (校准后的概率输出)   │
                    └─────────────────────┘
```

**优化项**：
```
HIST-01: 新建 analysis/historical_calibrator.py 模块
         — HistoricalCalibrator 类
         — 方法:
           · load_league_priors(league) → 按联赛加载历史先验
           · calibrate_spf(poisson_prob, league) → 贝叶斯校准SPF
           · calibrate_bf(score_probs, league) → 贝叶斯校准比分
           · calibrate_zjq(goal_probs, league) → 贝叶斯校准总进球
           · calibrate_bqc(bqc_probs, league) → 贝叶斯校准半全场
           
HIST-02: 在 analysis_pipeline.py 中集成 HistoricalCalibrator
         — 在 play_analyzer.analyze_all() 之后
         — 在 strategy_factory.analyze_all_plays() 之前
         — 插入校准步骤
         
HIST-03: 修复 lottery_historical_analysis 使用真实数据
         — 替换 hardcoded 模拟数据为 historical_tools 真实查询
```

## 四、架构重构优化

### 4.1 赔率键名统一

**问题**：每个玩法方法内部都有多套赔率键名兼容逻辑

**方案**：抽取统一的赔率键名标准化器

